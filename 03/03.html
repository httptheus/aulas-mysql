Curso MySQL #03 - Criando o Primeiro Banco de Dados

O modelo relacional

A sigla de SQL para "Structured Query Language", ou Linguagem de Consulta Estruturada, não carrega o nome atoa. Quando falamos em "bancos de dados estruturados", falamos em bancos de dados que funciona à partir de modelos relacionais de dados.

o modelo relacional e todos os elementos de modelagem de dados que conhecemos quando estudamos banco de dados estruturais, diz que um banco de dados relacional é constituído de tabelas que descrevem características em comum e que se relacionam com outras tabelas.

Em um exemplo prático, uma tabela no banco de dados referente ao cliente pode existir dentro deste contexto. E dentro da tabela de cliente podemos ter uma quantidade x de características que ajudam a identificar cada registro dentro desta tabela. Essas características podem incluir: o nome do cliente, a data de nascimento do cliente, o RG do cliente, etc. Um cliente pode se relacionar com outra tabela nomeada "Compra", que tem como características data da compra e hora da compra, por exemplo.

Note que este exemplo pode ser traduzido para o ambiente de banco justamente por fazermos essa abstração com o mundo real. Clientes podem realizar compras, assim como se faz fisicamente, e no modelo de banco de dados relacional, essa interação entre cliente e compra é chamada de relacionamento. Esse é o esquema de funcionamento de todo banco de dados baseado no modelo relacional.



E como criar um banco de dados?

"Com o comando CREATE DATABASE" é a resposta!

Mas antes de qualquer coisa, é necessário entender algumas outras coisas. Entre elas, as "Queries". Uma "Query", no contexto do banco de dados dentro do Workbench, é onde está o "código-fonte" do nosso banco de dados com SQL. Uma Query pode ser totalmente executada como um código normal, bem como pode ser executada linha-a-linha, individualmente. Cada execução, total ou em apenas trechos de código, é a execução de uma Query que vai retornar um valor especificado.

Isso significa que não necessariamente precisamos executar o mesmo trecho onde criamos o banco de dados para retornar valores de registro de uma tabela; podemos simplesmente executar a parte de comando em que se retorna o valor.

Dito isso, criemos o nosso banco de dados. Os comandos CREATE DATABASE é seguido pelo nome do banco de dados em si. Vamos criar um banco de dados de cadastro desta forma.


CREATE DATABASE bd_cadastro;
USE bd_cadastro;

O comando USE determina que todos os comandos de estruturação e manipulação do banco de dados serão feitos no banco de dados em questão. É necessário certificar se esta linha de comando foi devidamente executada.

Executando a linha de comando, o banco de dados com nome 'bd_cadastro' será criado. A partir daqui, será possível criar tabelas, definir suas características e criar os primeiros registros. Mas antes disso, vamos conhecer...



Os Tipos Primitivos em SQL

Tipos primitivos é um conceito primário e basico principalmente quando estamos falando de linguagens de programação, neste caso ainda mais quando falamos sobre bancos de dados e as informações a serem manipuladas dentro e fora dele.

No SQL, os dados primitivos com qual estamos acostumados se organizam e dividem-se entre famílias de tipos primitivos e suas subdivisões. Os quatro principais tipos primitivos do SQL são: Numéricos, Literais, Data/Tempo e os Espaciais. Os numéricos possuem os tipos de dados inteiro, real e lógico, que se subdividem em outros tipos primitivos, por exemplo.

Isso por quê bancos de dados SQL seguem pela linha de serem bastante precisos quando se trata da estruturização das tabelas e os dados a serem armazenados, levando em consideração a flexibilidade, consistência e robustez de um banco de dados. O comprimento de uma informação em bytes de qualquer tipo de dado dentro de um banco de dados pode ou não comprometer essas questões mencionadas levando em conta seu funcionamento, isso significa que temos vários tipos de valores do tipo 'Inteiro' dentro do SQL, entre eles o TINYINT, que especifica um valor de 3 algarismos entre 0 e 255, o SMALLINT entre os valores -32,768 e 32,767 e o INT, que vai do valor -2,147,483,648 a 2,147,483,647, e etc. etc. etc.

O grande ponto nisso tudo é a declaração correta para o tipo de informação proposta. Um valor inteiro TINYINT é mais do que o suficiente para especificar campos como 'idade' por exemplo. Isso se extende para tipos não numéricos onde o número de bytes é evidenciado.

A tabela com essas especificações em cada tipo pode ser encontrada aqui.

Dito isso, vamos conhecer os tipos de dados mais comumente utilizados dentro da estruturação dos campos de uma tabela em um banco de dados.

Tipo Descrição
VARCHAR Tipo primitivo que recebe um valor variável de caracteres. Esse valor variável é definido entre parênteses.
CHAR Tipo primitivo que define um valor fixo de caracteres. Esse valor é definido entre parênteses.
INT Tipo primitivo que define valores numéricos inteiros.
DECIMAL Tipo primitivo que define valores numéricos com vírgula.
DATE Tipo primitivo que especifica datas compostas pelo padrão AAAA-MM-DD.

... Entre vários outros tipos e seus subtipos.


Importante: VARCHAR determina que os valores abaixo ou igual ao que foi determinado em número de caracteres serão desconsiderados. Ao contrário de CHAR, onde o comprimento determinado deve ser satisfeito em sua totalidade. Exemplo: nomes são informações de comprimentos variáveis, ao mesmo tempo em que CPFs, por serem de comprimento igual a 11 caracteres são informações de comprimento fixo.



Criando uma tabela

Dito tudo isso, vamos definir a tabela "cliente", dentro do nosso banco de dados recém-criado.

À partir do mesmo comando que utilizamos para criar o banco de dados, vamos utilizar para criar uma tabela. Ou neste caso, uma TABLE.


CREATE DATABASE bd_cadastro;
USE bd_cadastro;

/*Criando uma tabela 'cliente' e seus campos.*/
CREATE TABLE cliente(
    nome VARCHAR(100),
    idade TINYINT,
    email VARCHAR(200),
    cpf CHAR(11)
);

A estrutura da tabela 'cliente' e qualquer outra no modelo relacional baseada em SQL é esta: você tem as diretivas de criação de uma tabela seguido pelo seu identificador. Dentro de parênteses, você tem os campos necessários que todo registro de cliente precisa ter armazenado dentro do banco. Cada campo/característica da tabela é identificado pelo seu nome + o tipo primitivo do dado a ser registrado: nome é do tipo VARCHAR de tamanho máximo de 100. Já cpf, por ser de comprimento fixo, recebe um char tamanho 11, etc.

Uma vez que a estrutura foi criada, experimente o comando abaixo:


...

DESCRIBE cliente;

Este comando vai abrir uma janela com todas as informações da estrutura detalhadamente. Nesta janela você consegue visualizar todos os campos e seus identificadores, tipos e suas propriedades.
