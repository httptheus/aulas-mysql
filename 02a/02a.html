Curso MySQL #02a - A Linguagem SQL

Structured Query Language

... Ou "SQL", é a línguagem utilizada para comunicação entre aplicação e servidor com banco de dados.

A linguagem SQL, por definição, subcategoriza seus comandos em outras definições específicas. O que signfica que o SQL por si só corresponde ao conjunto universo dos comandos que a compõe como linguagem. Para um melhor entendimento, vamos conhecer melhor esses comandos e a qual categoria eles se enquadram com base na característica de suas próprias funcionalidades:

  • DDL - ou "linguagem de definição de dados", correspondem aos comandos de definição de tabelas, informações e valores de cada tabela, bem como a estruturação do banco em si. Em resumo, são os comandos que interferem na estrutura do banco de dados.
  • DML - ou "linguagem de manipulação de dados", consiste nos comandos de manipulação de tabelas, o que se enquadram inserções de novos valores em tabelas; exclusões desses dados; alteração desses dados, etc.

Com isso, podemos resumir que a porção de comandos DDL refere-se a definição do banco de dados e suas tabelas, e os comandos DML são utilizados para trabalhar com a manipulação de dados dessas tabelas.

Além destes dois que entram como principais que compõe os fundamentos básicos acerca da linguagem SQL como um todo, também temos:

  • DQL - ou "linguagem de solicitação de dados", refere-se, principalmente, aos comandos de leitura e busca de informações registradas em um banco de dados.
  • DCL - ou "linguagem de controle de dados", são os comandos que abrangem a segurança e que definem níveis de acesso a um banco de dados: definir permissões de usuário ou quais comandos podem ser utilizados por tal usuário são algumas das coisas definidas pela DCL.
  • DTL - ou "linguagem de transação de dados, tratam-se dos comandos que especificam a transação dos dados uma vez que eles são solicitados.


Workbench, MySQL CLI...

O MySQL Workbench é uma IDE projetada para a linguagem SQL e também um SGBD para modelagem e teste de banco de dados. É uma alternativa mais amigável por possuir toda uma interface visual e componentes de uma IDE para o processo de definição e uso de banco de dados, que é o total oposto se essa atividade for feita através da CLI do MySQL, acessada pelo WAMP Server.

IMPORTANTE! A partir deste momento, durante o uso dessas aplicações, pode-se pedir nomes de usuário e senha para acesso aos bancos de dados. Por padrão, 'root' é o nome de usuário sem senha definida (definida literalmente por uma string vazia, ou '').

Isso significa que aprender SQL junto de uma interface visual parece melhor do que lidar com a tela monocromática e complicada do prompt de comando, mas não significa que deve ser descartado. Os comandos aprendidos podem ser postos em prática aqui também e funciona tão bem quanto a alternativa visual oferecida pelo Workbench.

No CLI do MySQL, ou MySQL Console, podemos executar vários comandos. Podemos tentar entrando com \h ou help para exibir uma lista de comandos de uso do próprio CLI.
